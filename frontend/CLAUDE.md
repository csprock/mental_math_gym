# User Interface

## UI Elements
There should be two pages of the user interface. 

1. The Practice "gym" where the user selects a Lesson type and number of practice Problems and then starts the timed practice session.
2. Progress Tracker - lets the user explore their progress on Lessons, Units and Problems. Includes graphics to chart progress over time.

## UI Conventions
### Practice Gym
* Problems that have parameters that can be set by the user should have consistent user-interface elements: boolean -> radio buttons, ints and floats -> text box, parameters with a limited set of elements -> dropdown menu. 
* Include an optional timer that starts with the practice session. 

### Progress Tracker
* Include a selection hierarchy that aggregates selections over the level of the hierarchy.
* Include a chart that shows the percentage of correct answers over time. 
* Include a table that shows Sessions and stats for sessions. 


## Misc Conventions
* feel free to add other UI elements that are compatible with the backend / API functionality that are not listed above.
